# tabilog 旅行記事一覧・詳細ページ 設計書

**Date:** 2026-03-18
**Project:** tabilog — 旅の記録ブログ
**Scope:** 記事一覧ページ (`app/page.tsx`) と詳細ページ (`app/posts/[id]/page.tsx`) の実装

---

## 概要

モックデータを使った旅行記事一覧ページと詳細ページを実装する。
一覧は日付降順でカード表示し、クリックで詳細ページへ遷移する。
デザインはセージグリーン (#4a7c59) で統一する。

---

## データモデル

`lib/posts.ts` にモックデータを集約し、両ページからインポートする。

```ts
type Post = {
  id: string        // URLに使用 (例: "kyoto-2025-autumn")
  title: string     // 記事タイトル
  date: string      // "YYYY-MM-DD" 形式
  location: string  // 場所 (例: "京都, 日本")
  thumbnail: string // Unsplash 旅行系画像の静的URL
  body: string      // 詳細ページ用ダミー本文
}
```

モック記事5件を日付降順でソートした配列 `posts` をエクスポートする。
`getPostById(id: string)` ヘルパーも同ファイルで提供する。

---

## 一覧ページ (`app/page.tsx`)

### レイアウト

- ページ上部にヘッダー: "tabilog" (h1) と "旅の記録" (サブタイトル)
- カードグリッド: デスクトップ2列、モバイル1列
- フッターなし（スコープ外）

### カードコンポーネント

各カードは `<Link href="/posts/[id]">` でラップされ、以下の要素を表示する:

1. サムネイル画像（`next/image`、アスペクト比 16:9）
2. 場所テキスト（小文字、セージグリーン）
3. タイトル（太字）
4. 日付（小テキスト、グレー）

ホバー時に軽いシャドウとスケールアップのトランジションを付ける。

### ソート順

`posts` 配列は `lib/posts.ts` 内で日付降順に並べておく（ページ側でのソート不要）。

---

## 詳細ページ (`app/posts/[id]/page.tsx`)

### ルーティング

- Next.js App Router の動的ルート: `app/posts/[id]/page.tsx`
- `params.id` で `getPostById()` を呼び出す
- 存在しない id は `notFound()` を返す

### レイアウト

1. 戻るリンク: "← 一覧へ" (`/` へのリンク、セージグリーン)
2. ヒーロー画像: 全幅、高さ 400px 固定、`object-cover`
3. メタ情報: 場所（セージグリーン）+ 日付（グレー）を横並び
4. タイトル (h1)
5. 本文: 2〜3段落のダミーテキスト

## デザインシステム

| 要素 | 値 |
|------|-----|
| テーマカラー | `#4a7c59` (セージグリーン) |
| フォント | システムデフォルト (日本語対応) |
| 背景 | ホワイト (`#ffffff`) |
| テキスト | ダークグレー (`#1a1a1a`) |
| サブテキスト | ミディアムグレー (`#6b7280`) |
| スタイル実装 | Tailwind CSS |

```
lib/

```

---

## 制約・前提

- 画像は Unsplash の旅行系画像の静的 URL を使用（ビルド時のフェッチなし）
- `next/image` を使用するため `next.config.ts` に Unsplash ドメインを追加する
- データ取得は将来 API/CMS に差し替え可能な構造にする
- TypeScript を使用する
